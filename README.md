## Build & Deployment

### Build Commands

#### Development Build
```bash
npm run dev
```

#### Production Build
```bash
npm run build
```

#### Start Production Server
```bash
npm run start
```

### Build Output Directory

After running `npm run build`, the generated files are located in:

```
.next/
```

This directory contains:
- **`.next/static/`** - Static assets (CSS, JS, images)
- **`.next/server/`** - Server-side code and API routes
- **`.next/cache/`** - Build cache files

### Deployment Paths

#### Local Development
- **URL**: `http://localhost:3000`
- **Build Output**: `.next/` directory

#### Production Deployment

For production deployment, you have several options:

1. **Vercel** (Recommended for Next.js)
   ```bash
   npm install -g vercel
   vercel
   ```
   - Automatic builds and deployments
   - Built-in CDN and edge functions
   - Environment variables management

2. **Docker**
   ```dockerfile
   FROM node:18-alpine
   WORKDIR /app
   COPY package*.json ./
   RUN npm ci --only=production
   COPY . .
   RUN npm run build
   EXPOSE 3000
   CMD ["npm", "start"]
   ```

3. **Traditional Server**
   - Build: `npm run build`
   - Start: `npm run start`
   - Default port: `3000`
   - Can be configured with `PORT` environment variable

4. **Static Export** (if configured)
   - Add `output: 'export'` to `next.config.mjs`
   - Build: `npm run build`
   - Output: `out/` directory
   - Deploy the `out/` directory to any static hosting service

### Production Environment Variables

Make sure to set these in your production environment:

```bash
CREEM_API_KEY=your_production_creem_key
APP_BASE_URL=https://qrcode.chdaoai.com
NODE_ENV=production
```

### Important Files for Deployment

- **`public/`** - Static files served from root (robots.txt, sitemap.xml, ads.txt, images)
- **`.next/`** - Build output (generated by `npm run build`)
- **`package.json`** - Dependencies and scripts
- **`.env.local`** - Environment variables (not committed to git)

## How It Works

- **QR Code Generation**: Uses `qrcode` library to generate QR code matrices, then custom SVG rendering for shapes and styling
- **Internationalization**: Uses `next-intl` for multi-language support
- **Subscription**: Integrates with Creem API for payment processing
- **Batch Processing**: Client-side CSV/Excel parsing and QR code generation

## Scripts

- `npm run dev` - Start development server

## Notes

- The build output directory `.next/` should not be committed to version control
- For production, ensure all environment variables are properly configured
- The application supports both server-side rendering and static generation
- SEO files (sitemap.xml, robots.txt) are in the `public/` directory and automatically served
